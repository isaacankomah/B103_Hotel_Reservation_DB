# Submission Checklist

Use this before uploading the final work.

## Repository

- [ ] GitHub repository link opens correctly.
- [ ] Repository is public or accessible to the marker.
- [ ] `README.md` explains the project clearly.
- [ ] `sql/01_schema.sql` is included.
- [ ] `sql/02_sample_data.sql` is included.
- [ ] `sql/03_queries.sql` is included.
- [ ] ER diagram source or image is included in `diagrams/`.
- [ ] Query outputs or screenshots are included.

## SQL Testing

- [ ] `01_schema.sql` runs successfully.
- [ ] `02_sample_data.sql` runs successfully after the schema.
- [ ] `03_queries.sql` queries run successfully.
- [ ] Screenshots show readable outputs.
- [ ] Update, insert, and delete examples are demonstrated.

## Report

- [ ] First page includes student name and student ID.
- [ ] First page includes GitHub repository link.
- [ ] First page includes video demonstration link.
- [ ] Introduction is written in your own words.
- [ ] System Design includes the ER diagram.
- [ ] Implementation explains tables, keys, and constraints.
- [ ] Results section includes query screenshots or outputs.
- [ ] Challenges and Solutions section is honest and specific.
- [ ] Conclusion includes future work.
- [ ] Report is under 3000 words.
- [ ] Report is exported as PDF.

## Video

- [ ] Video is between 3 and 5 minutes.
- [ ] Video shows the project files.
- [ ] Video explains the ER diagram.
- [ ] Video demonstrates key SQL queries.
- [ ] Video link is added to the report.

## Final Check

- [ ] No placeholder text remains, such as `[Add your video link here]`.
- [ ] No copied guide text remains unchanged.
- [ ] No repository updates are made after the submission deadline.
